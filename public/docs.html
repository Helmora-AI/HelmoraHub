Helmora AI HTTP API

Helmora AI exposes an OpenAI-compatible /v1 surface for tools and apps. Issue a hel_dev_ or hel_pro_ key in the admin SPA (Operate → API Keys), then call the endpoints below.

The admin Playground uses a separate session path (/api/chat/completions and does not consume these API keys.

Base URL

Use your Hub origin, for example:

http://127.0.0.1:20800

All paths in this document are relative to that origin.

Authentication

Every /v1/* request requires a Hub API key.

Headers

Authorization: Bearer hel_dev_…yourkey…
# or
x-api-key: hel_dev_…yourkey…

Key types

PrefixEnvNotes
hel_dev_devLocal / staging tools
hel_pro_proProduction integrations

Keys may have an optional USD budget and expiry. Plaintext is shown once at creation.

Never embed API keys in public frontends. Prefer server-side proxies.

GET /v1/models

Returns OpenAI-style { object: "list", data: [...] }.

Includes meta ids such as auto, mode/smart, provider default models, agents, and embedding placeholders (e.g. text-embedding-3-small).

curl -s http://127.0.0.1:20800/v1/models \
  -H "Authorization: Bearer $HELMORA_KEY"

Standard chat completions. Same auth as above.

Streaming (SSE)

Set "stream": true. Response:

  • Content-Type: text/event-stream
  • Frames: data: {…chat.completion.chunk…}
  • End: data: [DONE]
Admin Playground uses an enriched SSE contract (event: metadata). The public /v1 stream matches classic OpenAI chunk frames.

POST /v1/embeddings

OpenAI-compatible embeddings.

Request

{
  "model": "text-embedding-3-small",
  "input": "hello world"
}

input may be a string or an array of up to 64 strings. encoding_format must be float (default). Optional dimensions is forwarded when the upstream supports it.

Routing

  1. Try enabled OpenAI-compatible providers (/embeddings on their baseUrl).
  2. If none succeed, return deterministic demo vectors so smoke tests and offline Hub still work.

Response shape

{
  "object": "list",
  "data": [
    { "object": "embedding", "index": 0, "embedding": [/* floats */] }
  ],
  "model": "text-embedding-3-small",
  "usage": { "prompt_tokens": 3, "total_tokens": 3 }
}

Errors

HTTPtype (typical)When
401invalid_api_key / api_key_expiredMissing, wrong, disabled, or expired key
429insufficient_quotaKey budget exceeded
400invalid_request_errorBad body / validation
502 / 503upstream / no providersUpstream failure or nothing routable

Error bodies follow { "error": { "message": "…", "type": "…" } }.

Usage & billing

  • /v1 traffic increments the API key’s spentUsd and writes usage events with source: "api".
  • Playground traffic is metered as source: "admin_chat" and does not increase key spend.
  • Inspect rollups in the admin SPA under Tokens & Costs (filter by source).

Examples

Chat (JSON)

curl -s http://127.0.0.1:20800/v1/chat/completions \
  -H "Authorization: Bearer $HELMORA_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "messages": [{"role":"user","content":"Say hello in one short sentence."}]
  }'

Chat (SSE)

curl -N http://127.0.0.1:20800/v1/chat/completions \
  -H "Authorization: Bearer $HELMORA_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "auto",
    "stream": true,
    "messages": [{"role":"user","content":"Count to three."}]
  }'

Embeddings

curl -s http://127.0.0.1:20800/v1/embeddings \
  -H "Authorization: Bearer $HELMORA_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "model": "text-embedding-3-small",
    "input": ["alpha", "beta"]
  }'

JavaScript (fetch)

const res = await fetch("http://127.0.0.1:20800/v1/chat/completions", {
  method: "POST",
  headers: {
    Authorization: `Bearer ${process.env.HELMORA_KEY}`,
    "Content-Type": "application/json",
  },
  body: JSON.stringify({
    model: "auto",
    messages: [{ role: "user", content: "Ping" }],
  }),
});
const json = await res.json();
console.log(json.choices?.[0]?.message?.content);

Python

import os, requests

r = requests.post(
    "http://127.0.0.1:20800/v1/embeddings",
    headers={"Authorization": f"Bearer {os.environ['HELMORA_KEY']}"},
    json={"model": "text-embedding-3-small", "input": "hello"},
    timeout=60,
)
r.raise_for_status()
print(len(r.json()["data"][0]["embedding"]))

Limits & tips

  • JSON body size default: ~10 MB (Express).
  • Embeddings: max 64 inputs per request.
  • Prefer verified, enabled providers for real upstream quality; demo paths exist for offline smoke.
  • Point OpenAI SDKs at this Hub with base_url=http://127.0.0.1:20800/v1 and your Hub API key as api_key.
